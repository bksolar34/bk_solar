## E) 일조 시간·음영 영향 분석 -- 손실률 계산 프레임워크로 확장
"부지 공간 정보 설정" 화면에 새 패널 "발전량 손실률 계산(지형·이격거리·
기상 종합)"을 추가했습니다.
- 종단(전후열)/횡단(좌우열) 분석을 탭으로 전환할 수 있습니다.
- 지형 경사 방향(남향/북향/평지)과 경사도(°)를 입력하면 지평선
  장애물 각도에 가감되어 손실률에 반영됩니다(지구 자전축에 의한
  태양 입사각 자체는 기존 solarAltitudeDeg가 이미 연중 변화를
  실제 계산하고 있습니다).
- 전후/좌우 이격거리(m)와 발전설비 지상고(m)를 입력하면 어레이 간
  자기 그림자(self-shading) 손실을 임계 고도각(atan(높이/이격거리))
  근사식으로 계산합니다 -- 이격거리 조합을 바꿔가며 손실률이 어떻게
  달라지는지 바로 확인할 수 있습니다.
- 기상자료(Open-Meteo)의 구름량을 6시간 단위(00/06/12/18시)로
  샘플링해 평균 구름 손실을 추정치로 반영합니다.
- 하단부 용도(미사용/영농형/방목형)를 선택하면, 영농형일 때 작물에
  나눠주는 광투과 비율만큼 추가 손실로 반영됩니다.
- 모든 요인을 합산한 "추정 총 손실률"을 보여주되, 정밀 LIDAR/BIM
  시뮬레이션을 대체하는 것이 아니라 근사 추정 프레임워크임을 화면에
  명시했습니다.

## B) OpenStreetMap 3D 부지 뷰 -- 건물뿐 아니라 지형도 3D로
기존 3D 뷰는 OpenFreeMap의 fill-extrusion(건물)만 반영되어 있었고,
지형(경사/고도)은 평평했습니다. AWS Open Data Terrain Tiles(무료,
API 키 불필요, Terrarium 인코딩 PNG-DEM)를 raster-dem 소스로 추가하고
MapLibre의 setTerrain()을 호출해, 건물과 함께 지형 고저차도 실제
값 그대로 솟아오르게 했습니다. 2D 대안(표준 OSM/CyclOSM)에는 지형을
걸지 않습니다(평면 지도 위에서 지형 왜곡은 가독성만 떨어뜨리므로).

## 1) 출력제어/역률/주파수변경에 "적용" 확인 절차 추가
그동안 발전소 원격 제어 화면에서 출력제어(%)/역률(%)/주파수변경(Hz)은
숫자를 바꾸는 즉시 저장되어버려서 실제 현장 제어 흐름(값 확인 →
적용 → 반영)과 맞지 않았습니다. 이번에 매뉴얼과 동일한 흐름으로
바꿨습니다.
- 숫자를 바꾸면 일단 "임시값(초안)"으로만 남고, 아직 적용되지 않은
  행에는 주황색 "미적용" 표시가 붙습니다.
- [적용] 버튼을 눌러야 확인창("~설정을 적용하시겠습니까?")이 뜨고,
  확인을 눌러야 실제 값이 반영됩니다.
- [제어 결과]는 항상 "적용된"(확정) 값 기준으로 산출되므로, 적용을
  누르지 않고 숫자만 바꾼 상태로는 결과가 바뀌지 않습니다.

## 2) 인버터별 개별 차단기 추가
"발전소 제어" 패널에 기존 메인 차단기/인버터 차단기(전체) 외에,
인버터 하나하나를 따로 켜고 끌 수 있는 "인버터별 개별 차단기" 목록을
추가했습니다. (기존에는 개별 ON/OFF가 "인버터 개별 제어" 표 안에
출력/역률/주파수 설정과 뒤섞여 있어서 눈에 잘 안 띄었습니다 -- 이제
차단기 조작과 운전 조건 설정을 화면에서 분리했습니다.)

## 3) 부지 공간 정보 - 실제 3D 건물 뷰로 교체
기존 "3D"는 사실 2D 지도에 카메라만 기울인 것이었습니다. 이번에
OpenFreeMap(무료, API 키 불필요)의 벡터 타일 스타일로 교체해서, 건물
하나하나가 실제 높이로 표시되는 진짜 3D 뷰가 되도록 했습니다. 이
레이어가 기본값이고, 기존 표준 OSM/CyclOSM은 2D 대안으로 남아 있습니다.

## 4) 일조 시간·음영 분석 화면 직관적으로 재설계
"3D 지형 및 음영 분석"이라는 이름과 실제 내용(태양 고도 계산)이
안 맞아서 이해하기 어려웠던 점을 고쳤습니다.
- 이름을 "일조 시간 및 음영 영향 분석"으로 바꾸고, 위에 이 화면이
  무엇을 보여주는지 한두 문장으로 설명을 넣었습니다.
- "장애물 고도각 35°" 같은 추상적인 숫자 대신 "고층 건물/언덕 수준"
  처럼 사람이 바로 이해할 수 있는 설명을 함께 보여줍니다.
- 막대 높이로 표현하던 차트를 시간대별 색상 타임라인(정상 일사=주황,
  가려짐=빨강, 야간=회색)으로 바꾸고 범례를 추가해 한눈에 파악할 수
  있게 했습니다.
- 결과도 "오늘 낮 8시간 중 25%(2시간)가 가려질 것으로 예상됩니다"처럼
  평문장으로 요약합니다.

## 5) 코드 점검 (중복/오류 수정)
- 부지 공간 정보 설정 화면에서 `handleGoogleApiKeyChange` 함수가
  실수로 두 번 선언되어 있던 것을 발견해 제거했습니다.
- 전체 소스에서 함수/컴포넌트 중복 선언, NAV_TREE·DETAIL_PAGES·
  STUB_NOTES의 라우트 키 중복 여부를 스크립트로 재검사해 이상 없음을
  확인했습니다.
- 삭제된 "현장 시험 체크리스트" 기능과 관련된 사용되지 않는 Store
  메서드(getProtectionTests/setProtectionTestResult)와 데이터
  (PROTECTION_TEST_DEFAULTS)를 완전히 제거해 죽은 코드를 정리했습니다.
- Babel 트랜스파일 검증과 가상 DOM 스모크 테스트(엔진 파일 로드
  성공/실패 두 시나리오)로 문법·참조 오류가 없는지 재확인했습니다.
